Merging & Rebasing

To bring changes from one local branch into another you have two options: merge or rebase. Tower allows you to update your current HEAD branch using either strategy.

When doing a merge, a new commit (a so-called "merge" commit) will be created on top of both branches. During a merge, none of the existing commits are altered in any way.

When doing a rebase, Git takes all the committed changes from one branch and reapplies them (like patches) on the other branch one after the other. Thereby, history is rewritten: It will appear as if commits would have been made consecutively on the same branch, not on parallel branches. No merge commit will be created, but commits will have new SHA1-IDs.

IMPORTANT:As a golden rule you should never rebase commits that have already been published on a remote repository (pushed).

To merge a local branch into your HEAD branch

Perform one of the following actions:

  • In the Sidebar drag the local branch whose changes you want to integrate and drop it onto your HEAD branch.

  • Click the "Merge" toolbar button and select from the following menu which local branch you want to integrate with your HEAD branch.

To rebase a local branch into your HEAD branch

Perform one of the following actions:

  • In the Sidebar drag the local branch whose changes you want to integrate and drop it onto your HEAD branch, while keeping the ALT or CMD key pressed.

  • Click the "Rebase" toolbar button and select from the following menu which local branch you want to integrate with your HEAD branch.

NOTE:Tower will always integrate changes from another branch into your current HEAD branch, not the other way around.
